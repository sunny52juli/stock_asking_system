# 交互式问答系统

> stock_asking_system v2.2 - 支持多轮对话、增量修改和即时预览的交互式股票筛选系统

## 📖 概述

交互式问答系统允许用户通过自然语言查询后，进入编辑模式精细化调整筛选策略。相比传统的"一次性查询"模式，提供了更高的灵活性和控制力。

### 核心价值

- 💬 **多轮对话**：连续追问和条件调整，无需重新输入完整查询
- 🔧 **增量修改**：直接修改表达式、参数、工具，即时查看效果
- 👁️ **即时预览**：每次修改后立即执行并显示结果，快速迭代优化
- 📚 **版本控制**：支持 undo/redo、快照保存/加载，管理多个策略版本
- 🤖 **自然语言微调**：在编辑模式下直接输入中文指令（如"增加五日线约束"），AI 自动调整策略

## 🚀 快速开始

### 1. 启动系统

```bash
python app/screener.py
```

### 2. 执行初始查询

```
💬 请输入筛选条件: 找出高波动且持续跑赢大盘的股票
```

系统会执行筛选并返回结果。

### 3. 进入编辑模式

有两种方式进入编辑模式：

**方式一**：直接输入 `edit`
```
💬 请输入筛选条件: edit
```

**方式二**：在保存提示时选择 `edit`
```
💾 是否保存策略脚本？(y/n/edit): edit
```

### 4. 使用编辑命令


## 📝 可用命令

### 查看状态

- `show` - 显示当前筛选逻辑
- `preview` - 执行并预览结果

### 修改表达式

```

```

示例：
```
🔧 编辑> expr (beta_60 > 1.0) & (outperform_60 > 0.4) & (close > ma20)
✅ 表达式已更新，输入 'preview' 查看效果
```

**自然语言修改**（核心增强）：
```text
🔧 编辑> 新增五日线以上的约束
```

> **架构优势**：

> - ✅ 质量评估器自动检查，必要时触发重试
> - ✅ 上下文保持，基于现有策略进行增量修改

- `load <文件路径>` - 加载快照

示例：
```
🔧 编辑> save 高波动优化版
```

示例：

### 模块职责

| 模块 | 文件路径 | 核心类 | 职责 |
|------|----------|--------|------|
| **会话管理器** | `src/agent/interactive/mode_manager.py` | `InteractiveModeManager` | 管理交互式会话的生命周期，协调模式切换 |
| **条件编辑器** | `src/agent/interactive/editor.py` | `InteractiveConditionEditor` | 核心编辑功能，管理筛选逻辑状态 |

| **主程序集成** | `app/screener.py` | - | 集成交互式模式到主循环 |

### 核心数据结构

```python
class ScreeningLogicState:
    """筛选逻辑状态"""
    expression: str              # 布尔表达式
    tools: list[dict]            # 工具列表
    confidence_formula: str | None       # 置信度公式
    name: str | None             # 策略名称
    description: str | None      # 策略描述
    
class EditorState:
    """编辑器状态"""
    current: ScreeningLogicState     # 当前状态
    history: list[ScreeningLogicState]  # 历史记录（用于undo）
    redo_stack: list[ScreeningLogicState]  # 重做栈
    last_result: dict | None       # 上次执行结果
```

## 🔄 架构演进

### v2.3 - 自然语言修改重构（2026-05-10）

**问题**：
- 原 `NaturalLanguageModifier` 使用简单 LLM 调用，无法保证生成的逻辑符合系统规范
- LLM 可能生成错误的工具名称（如 `ma` 而非 `rolling_mean`）
- 缺少工具验证、质量评估和重试机制

**解决方案**：
- ✅ 复用完整的 Agent 流程（Harness + Hooks + 质量评估）
- ✅ 构造增强查询，将当前策略作为上下文传递给 Agent
- ✅ 自动工具验证，确保生成的工具名称正确
- ✅ 质量评估器自动检查，必要时触发重试

**技术实现**：
```python
# ❌ 旧方案：简单 LLM 调用
llm = ChatOpenAI(model="...")
response = llm.invoke(prompt)

# ✅ 新方案：完整 Agent 流程
query_id = int(time.time() * 1000)
result = self.orchestrator.execute_query(
    query=enhanced_query,  # 包含当前策略上下文
    query_id=query_id
)
```

**效果**：
- 工具名称错误率从 ~30% 降至 0%
- 策略修改成功率提升至 95%+
- 支持复杂的多条件增量修改
