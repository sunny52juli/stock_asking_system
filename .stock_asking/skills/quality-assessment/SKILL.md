# 质量评估与重试策略

## 📊 候选数量评估

| 候选数量 | 评分 | 说明 | 建议操作 |
|---------|------|------|----------|
| 0 | 0.0 | 无候选，完全失败 | 放宽条件重试 |
| 1-5 | 0.4 | 过少，可能条件太严 | 适当放宽阈值 |
| 6-9 | 0.6 | 偏少，可接受 | 可选择重试 |
| **10-30** | **1.0** | **最优区间** | ✅ 直接返回 |
| 31-50 | 0.7 | 偏多，建议增加过滤 | 可增加条件 |
| >50 | 0.3 | 过多，条件太松 | 必须收紧条件 |

## 🏭 行业分散度评估

```
行业分散度 = 不同行业数量 / min(候选总数, 10)
```

- **>0.7**：优秀（1.0分）- 行业分布均匀
- **0.5-0.7**：良好（0.7分）- 可接受
- **<0.5**：较差（0.4分）- 行业集中度过高

## 📈 回测指标评估（如有）

- **Sharpe Ratio > 0**：正向收益风险比（+0.3分）
- **最大回撤 < 30%**：风险控制良好（+0.3分）
- **胜率 > 50%**：策略有效性强（+0.2分）

## 🎯 综合质量分数计算

```python
quality_score = (
    候选数量评分 * 0.4 +
    行业分散度评分 * 0.2 +
    回测指标评分 * 0.2 +
    筛选条件合理性 * 0.2
)
```

### 决策规则

- `quality_score >= 0.7`：**通过**，返回结果给用户
- `0.5 <= quality_score < 0.7`：**警告**，但仍返回结果
- `quality_score < 0.5`：**不通过**，调整条件后重试

## 🔄 智能重试策略

### 重试触发条件

1. **候选数量为 0**：立即重试
2. **质量评分 < 0.5**：建议重试
3. **参数验证失败**：自动重试（最多3次）

### 重试调整策略

| 问题类型 | 调整策略 | 示例 |
|---------|---------|------|
| 候选过少 | 放宽阈值 | `rsi > 30` → `rsi > 25` |
| 候选过多 | 收紧条件 | 增加行业限制 |
| 参数错误 | 修正参数 | `window=100` → `window=60` |
| 无结果 | 减少约束 | 移除次要条件 |

### 重试次数限制

- **最大重试次数**：3次
- **连续错误限制**：2次
- **超过限制**：停止重试，返回最佳结果并说明原因

## 💡 实用建议

### 当结果为空时

1. 检查表达式是否过于严格（如同时要求多个极端条件）
2. 确认工具阈值是否在合理范围内（参考 tool-value-ranges）
3. 验证是否使用了不存在的字段或变量
4. 尝试逐步放宽条件，找到平衡点

### 当结果过多时

1. 增加行业限制（如只选"科技"、"医药"）
2. 提高阈值要求（如 `volatility < 0.2` → `volatility < 0.15`）
3. 添加额外过滤条件（如市值范围、市盈率等）

### 当质量评分低时

1. 查看具体的 issues 和 suggestions
2. 优先解决评分最低的维度
3. 参考历史成功案例调整策略
