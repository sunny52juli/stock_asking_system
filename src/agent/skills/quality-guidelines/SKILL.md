# 质量评估指南

## 候选数量评估

### 理想范围
- **1-50 个**: 最佳范围，便于深入分析
- **50-100 个**: 可接受，但建议适当收紧条件

### 需要调整
- **0 个**: verdict=retry
  - 建议: 放宽条件、减少指标约束、扩大行业范围
- **>100 个**: verdict=retry
  - 建议: 增加筛选条件、提高阈值、缩小行业范围

## 工具执行成功率

### 评估标准
- **全部成功**: 继续评估其他维度
- **部分失败**: verdict=retry
  - 检查工具参数是否正确
  - 确认数据格式是否符合要求
- **全部失败**: verdict=fail
  - 停止重试，返回错误信息

## 指标多样性

### 评估原则
- **单一指标**: 风险较高，容易过拟合
- **2-3个指标**: 理想组合，平衡效果与复杂度
- **>5个指标**: 可能过度拟合，建议简化

### 指标类别平衡
建议包含不同类别的指标:
- 趋势类 (MACD, EMA)
- 超买超卖类 (RSI, KDJ)
- 成交量类 (OBV, Volume Ratio)
- 波动率类 (ATR, Bollinger)

## 质量评分计算

```python
score = (
    candidate_count_score * 0.3 +    # 候选数量合理性
    tool_success_rate * 0.3 +        # 工具执行成功率
    indicator_diversity * 0.4         # 指标多样性
)
```

### 候选数量评分
- 50个候选: 1.0分
- 25个候选: 0.5分
- 100个候选: 1.0分
- 0个或>100个: 0.0分

### 工具成功率评分
- 全部成功: 1.0分
- 80%成功: 0.8分
- <50%成功: 0.0分

### 指标多样性评分
- 3个不同类别指标: 1.0分
- 2个指标: 0.67分
- 1个指标: 0.33分

## Retry 建议模板

### 候选过少
- "放宽 RSI 阈值从 30 提高到 40"
- "移除成交量条件"
- "扩大到全市场而非单一行业"

### 候选过多
- "增加涨幅条件: 日涨幅 > 3%"
- "添加 RSI 上限: RSI < 70"
- "限定到特定行业"

### 工具失败
- "检查 rsi 参数: window 应为整数"
- "确认 pct_change 的 periods 参数"
- "验证数据列名是否正确"
